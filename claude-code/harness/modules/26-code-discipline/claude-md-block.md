### Code discipline (26-code-discipline)

Code discipline: naming/logging/error-handling/SQL-safety conventions, lean-code
principles (small functions, no speculative abstraction), the deletion test for new
modules (`docs/harness/26-code-discipline.md`), and test-first via
`superpowers:test-driven-development`.
